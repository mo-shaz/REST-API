CRUD API

Implemented on a Login and Register System

API End-points and Instructions

    > POST (CREATE) to https://shellcorp-api.herokuapp.com/api/register to create an account in the form:
    { "username": name, "email": mail, "password": password }
    > POST (READ) to https://shellcorp-api.herokuapp.com/api/login to login to the created account and receive an AuthToken, request form:
    { "email": your_name, "password": your_password }
    > Copy the given AuthToken in the form: BEARER /AUTH_TOKEN/ in order to access the user details.
    > PUT (UPDATE) to https://shellcorp-api.herokuapp.com/api/update with the given AuthToken in the header field to change/update either the username or password of the account or both. The request body should be in the form:
    { "username": new_name, "password": new_password }
    > DELETE Method require Admin privilege. To do that, just login using the credentials
    { "email": "admin@shell.com", "password": "adminhere" }
    Now you can use the DELETE method on the url https://shellcorp-api.herokuapp.com/delete with the received AuthToken in the authorization field of the header to delete any account by composing the request body in the form
    { "email": target_mail }
    > You can also GET (ADMIN READ) to https://shellcorp-api.herokuapp.com/users with the recieved admin AuthToken to access the details of every user in the database.
