CRUD API

implemented on a Login and Register System


API End-points and Instructions

POST (CREATE) to https://shellcorp-api.herokuapp.com/api/register to create an account in the form:

{
"username": name,
"email": mail,
"password": password
}


POST (READ) to https://shellcorp-api.herokuapp.com/api/login to login to the created account and receive an AuthToken, request form:

{
"email": your_name,
"password": your_password
}

Copy the given AuthToken in the form: BEARER /AUTH_TOKEN/. This is needed for later use.


PUT (UPDATE) to https://shellcorp-api.herokuapp.com/api/update with the given AuthToken in the header field to change/update either the username or password of the account or both. The request body should be in the form:

{
"username": new_name,
"password": new_password
}


DELETE Method require Admin privilege. To do that, just login on https://shellcorp-api.herokuapp.com/api/login using the credentials:

{
"email": "admin@shell.com",
"password": "adminhere"
}

Use DELETE method on the url https://shellcorp-api.herokuapp.com/api/users/target-email with the received admin AuthToken in the authorization field of the header. Make sure to add the complete mail address on the target-email field.


You can also use GET method (ADMIN READ) to https://shellcorp-api.herokuapp.com/api/users with the recieved admin AuthToken to access the details of every user in the database. Further, you can access single user details by specifying their email in the URL in the form: https://shellcorp-api.herokuapp.com/api/users/target-email
